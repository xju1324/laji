# 周报PPT制作指南

## 快速填写模板变量

### 需要填写的变量

| 变量 | 建议值 | 说明 |
|------|--------|------|
| [X] | 1 | 迭代开始周数 |
| [X+2] | 3 | 迭代结束周数 |
| [项目名称] | 垃圾识别管理系统 | 项目正式名称 |
| [软件功能] | 智能垃圾识别管理 | 系统核心功能描述 |
| [X]% (整体进度) | 30% | 项目整体完成度 |
| [X]% (迭代进度) | 30% | 本次迭代完成度 |
| [您的姓名] | 您的真实姓名 | 报告人姓名 |
| [Git仓库地址] | 您的Git URL | 项目仓库地址 |

## PPT页面结构建议

### 必须包含的页面（核心15页）

1. **封面** - 项目基本信息
2. **项目概述** - 愿景和目标
3. **迭代周期与进度** - 时间线和进度
4. **本次迭代目标** - 用户故事完成情况
5. **技术架构** - 技术栈介绍
6. **核心功能实现** - 功能清单
7. **数据库设计** - 表结构说明
8. **代码统计** - Git统计信息
9. **遇到的挑战** - 问题与解决
10. **测试情况** - 测试结果
11. **Git提交记录** - 提交历史
12. **下一步计划** - 下阶段任务
13. **风险与问题** - 风险识别
14. **项目亮点** - 技术亮点
15. **总结** - 迭代总结

### 可选页面（根据实际情况）

- 团队介绍
- 项目时间线
- 功能演示截图
- 性能指标
- 用户反馈

## 设计建议

### 配色方案
- **主色调：** 绿色系（#4CAF50, #8BC34A）- 环保主题
- **辅助色：** 蓝色系（#2196F3）- 科技感
- **强调色：** 橙色系（#FF9800）- 重要信息

### 图表建议

1. **进度展示**
   - 使用进度条或饼图
   - 标注完成百分比
   - 使用不同颜色区分

2. **功能模块**
   - 使用流程图或架构图
   - 标注已完成模块
   - 使用图标增强视觉效果

3. **时间线**
   - 使用甘特图或时间轴
   - 标注关键里程碑
   - 显示当前进度

## 演示技巧

### 时间分配
- **开场介绍：** 2分钟（封面+概述）
- **核心内容：** 10分钟（功能+技术+进度）
- **问题与计划：** 5分钟（挑战+计划）
- **总结：** 2分钟
- **Q&A：** 3-5分钟

### 重点强调
1. **已完成功能** - 用✅标记
2. **技术亮点** - 突出技术创新
3. **进度达成** - 用数据说话
4. **问题解决** - 展示解决问题的能力

### 常见问题准备
- "为什么选择这个技术栈？"
- "遇到了什么最大的挑战？"
- "下一步的计划是什么？"
- "预计什么时候能完成？"
